Hablémos de software
En esta sección les hablaré sobre el panorama general de la industria del software, tecnologías, tendencias, consejos laborales y algunos casos de estudio que he tenido la oportunidad de conocer.
Introducción a la programación
Aquí hablaremos de la programación desde un punto de vista técnico e historico, hablarémos de python como lenguaje de programación y sus características.
Conociendo la terminal de comandos
Les mostraré como usar la terminal de comandos de windows para las tareas básicas que necesitaremos realizar durante el desarrollo, como crear directorios y desplazarse entre ellos y ejecutar python en modo interactivo o mediante archivos fuente.
Entrada y salida de datos
Les mostraré PRINT e INPUT, dos funciones de ENTRADA/SALIDA de python que usaremos a lo largo del curso.
Variables para almacenar datos y algo más
También hablaremos de las variables. tipos de datos y sus conversiones.
Toda clase de operadores
Aprenderán el uso de diferentes clases de operadores: aritméticos, lógicos, de nivel bit y algunos ejemplos al respecto.
Controlando el flujo del programa
En este punto comenzaremos a dirigir la toma de desiciones de nuestro programa por medio de condicionales.
Listas de datos y como usarlas
Entonces descubriremos las listas de datos y los ciclos.
Funciones para todo
Después, a definir funciones y cómo usarlas para ahorrar código.
Estructuras, más allá de las listas
También estudiaremos algunos tipos de datos complejos, como los diccionarios y las tuplas, e incluso sus combinaciones con listas y variables simples.
Módulos propios y ajenos
Y finalmente aprenderán como divivir el código y prepararlo para que pueda ser reutilizado en múltiples proyectos.

¿Qué es un lenguaje de programación?

Un lenguaje de programación es un lenguaje formal que especifica una serie de instrucciones para que una computadora produzca diversas clases de datos. Los lenguajes de programación pueden usarse para crear programas que pongan en práctica algoritmos específicos que controlen el comportamiento físico y lógico de una computadora.

¿Cuantos lenguajes de programación existen?

El Zen de python

Bello es mejor que feo.
Explícito es mejor que implícito.
Simple es mejor que complejo.
Complejo es mejor que complicado.
Plano es mejor que anidado.
Disperso es mejor que denso.
La legibilidad cuenta.
Los casos especiales no son tan especiales como para quebrantar las reglas.
Lo práctico gana a lo puro.
Los errores nunca deberían dejarse pasar silenciosamente.
A menos que hayan sido silenciados explícitamente.
Frente a la ambigüedad, rechaza la tentación de adivinar.
Debería haber una -y preferiblemente sólo una- manera obvia de hacerlo.
Aunque esa manera puede no ser obvia al principio a menos que usted sea holandés.
​
Ahora es mejor que nunca.
Aunque nunca es a menudo mejor que ya mismo.
Si la implementación es difícil de explicar, es una mala idea.
Si la implementación es fácil de explicar, puede que sea una buena idea.
Los espacios de nombres (namespaces) son una gran idea ¡Hagamos más de esas cosas!
"Holandés" hace referencia a Guido van Rossum, el autor del lenguaje de programación Python, que es holandés. También hace referencia a la gran concentración de desarrolladores holandeses conocidos en relación a otras nacionalidades.
